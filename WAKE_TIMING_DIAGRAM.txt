/*
 * ===========================================================================
 * BQ79616 WAKE SEQUENCE TIMING DIAGRAM
 * ===========================================================================
 *
 * NORMAL STATE (Before Wake):
 * ============================
 * Device is in SHUTDOWN mode, waiting for TX-low pulses to wake
 *
 *
 * TIMELINE OF WAKE SEQUENCE:
 * ==========================
 *
 * t=0ms      t=2.75ms     t≈6.25ms    t≈9.75ms    t≈13.25ms    t≈24.8ms
 *  |            |           |           |           |            |
 *  V            V           V           V           V            V
 *
 * START        PULSE1       PULSE2      UART        UART    PROPAGATION
 * HIGH         LOW→HIGH     LOW→HIGH    REINIT      SEND    DONE
 *                                       READY       WAKE    
 *
 * ┌──────────────────────────────────────────────────────────────────────────┐
 * │                                                                            │
 * │ TX LINE STATE:                                                            │
 * │                                                                            │
 * │   HIGH+                                                                   │
 * │   ────────────────────────────────────────────────────────────────      │
 * │      │    └─────┐        └─────┐                                         │
 * │      │           │             │                                         │
 * │   LOW+            └─────────────┘                                        │
 * │                                                                            │
 * │ DETAIL: PULSE 1                        DETAIL: PULSE 2                   │
 * │ ───────────────                        ──────────────                    │
 * │                                                                            │
 * │   START: TX=HIGH (idle)                 START: TX=HIGH (after settle)   │
 * │   DURING: TX driven LOW for 2.75 ms     DURING: TX driven LOW for 2.75ms
 * │   RELEASE: TX returns to HIGH            RELEASE: TX returns to HIGH    │
 * │                                                                            │
 * │   Then: Wait 3.5 ms (shutdown interval)  Then: Wait 3.5 ms (bridge ACTIVE
 * │                                                                            │
 * └──────────────────────────────────────────────────────────────────────────┘
 *
 *
 * DETAILED TIMELINE WITH CODE ACTIONS:
 * =====================================
 *
 * TIME    | CODE ACTION                  | DEVICE STATE            | TX PIN STATE
 * --------|------------------------------|------------------------|------------------
 * 0 ms    | HAL_UART_DeInit()           | Preparing to wake       | Will be released
 * 0 ms    | bq_pin_tx_to_gpio()         | UART control released   | Idle HIGH
 * 0 ms    | bq_pin_tx_set(HIGH)         | Ready for pulse         | HIGH (idle)
 * --------|------------------------------|------------------------|------------------
 * 0 ms    | bq_pin_tx_set(LOW)          | WAKE PULSE 1 START      | LOW (held)
 * 2.75 ms | bq_pin_tx_set(HIGH)         | WAKE PULSE 1 END        | HIGH
 * --------|------------------------------|------------------------|------------------
 * 2.75 ms | bq_delay_us(3500)           | Shutdown settling time  | HIGH (idle)
 * 6.25 ms | bq_delay_us() complete      | Ready for pulse 2       | HIGH
 * --------|------------------------------|------------------------|------------------
 * 6.25 ms | bq_pin_tx_set(LOW)          | WAKE PULSE 2 START      | LOW (held)
 * 9.00 ms | bq_pin_tx_set(HIGH)         | WAKE PULSE 2 END        | HIGH
 * --------|------------------------------|------------------------|------------------
 * 9.00 ms | bq_delay_us(3500)           | Bridge entering ACTIVE  | HIGH
 * 12.5 ms | bq_delay_us() complete      | Allow UART to stabilize | HIGH
 * --------|------------------------------|------------------------|------------------
 * ~13 ms  | bq_pin_tx_to_uart()         | UART control restored   | HIGH (under UART)
 * ~13 ms  | HAL_UART_Init()             | UART ready to TX        | HIGH
 * ~13 ms  | HAL_Delay(10)               | Settle time             | HIGH
 * --------|------------------------------|------------------------|------------------
 * ~23 ms  | bq79616_write()             | SEND UART FRAME         | DATA BYTES
 * ~23 ms  | [CONTROL1 SEND_WAKE write]  | Device receives command | UART TRAFFIC
 * --------|------------------------------|------------------------|------------------
 * ~24 ms  | bq_delay_us(11600 * N)      | Stack propagation wait  | HIGH (idle)
 * ~35 ms  | bq_delay_us() complete      | Stacked devices woken   | HIGH
 * --------|------------------------------|------------------------|------------------
 * ~35 ms  | Function returns (rc=0)     | DEVICE IS NOW ACTIVE    | HIGH
 * --------|------------------------------|------------------------|------------------
 *
 *
 * WHAT EACH PHASE DOES:
 * =====================
 *
 * 1. PULSE 1 (0-2.75 ms)
 *    Purpose: Initial SHUTDOWN-to-ACTIVE transition signal
 *    Device sees: TX line held LOW for 2.75 ms
 *    Effect: Device wakes from SHUTDOWN to ACTIVE
 *    Critical: Must be exactly 2.75 ms (±5%)
 *
 * 2. SHUTDOWN INTERVAL (2.75-6.25 ms)
 *    Purpose: BQ79616 internal synchronization window
 *    Device sees: TX released, returns to idle
 *    Effect: Device stabilizes in ACTIVE mode
 *    Critical: Must be ≥3.5 ms before next pulse
 *
 * 3. PULSE 2 (6.25-9.00 ms)
 *    Purpose: Confirmation/re-sync pulse for stacking
 *    Device sees: TX line held LOW again for 2.75 ms
 *    Effect: Device confirms it received wake signal
 *    Critical: Timing between pulses validated in step 2
 *
 * 4. BRIDGE ACTIVE WAIT (9.00-12.5 ms)
 *    Purpose: Allow bridge to fully enter ACTIVE state
 *    Device sees: TX idle after second pulse
 *    Effect: Analog power supply transitions on
 *    Critical: ≥3.5 ms allows power to stabilize
 *
 * 5. UART RESTORATION (12.5-13+ ms)
 *    Purpose: Switch TX pin from GPIO to UART alternate function
 *    Device sees: TX under UART control again
 *    Effect: Device ready for command frames
 *    Critical: Must wait for UART peripheral to init
 *
 * 6. SEND_WAKE COMMAND (13-23 ms)
 *    Purpose: Propagate wake signal to stacked BQ79616 devices
 *    Device sees: UART frame with CONTROL1[SEND_WAKE]=1
 *    Effect: Stacked devices also wake from SHUTDOWN
 *    Critical: Only needed if you have multiple BQ79616 in stack
 *
 * 7. STACK PROPAGATION (23-35 ms)
 *    Purpose: Wait for wake to propagate through stack
 *    Device sees: Nothing (master waits)
 *    Effect: All stacked devices in ACTIVE state
 *    Critical: 11.6 ms × (number of devices)
 *
 *
 * WHAT CAN GO WRONG:
 * ==================
 *
 * ❌ PROBLEM: Pulse widths are 0 ms
 *    Cause: GPIO pin not configured as output
 *    Fix: Check bq_pin_tx_to_gpio() actually running
 *    Evidence: TX never goes LOW on oscilloscope
 *
 * ❌ PROBLEM: Pulse widths are ~500 ms or ~5000 ms
 *    Cause: SystemCoreClock value is wrong
 *    Fix: Verify correct clock speed in system setup
 *    Evidence: Pulse width is off by factor of N
 *
 * ❌ PROBLEM: TX pin never returns to HIGH after pulse
 *    Cause: GPIO.PIN_SET not working (hardware issue)
 *    Fix: Check GPIO configuration, pull resistors, hardware
 *    Evidence: TX line stuck LOW on oscilloscope
 *
 * ❌ PROBLEM: SEND_WAKE write fails (rc != 0)
 *    Cause: Device not responding to UART (still in SHUTDOWN)
 *    Fix: Check pulses actually worked, verify UART baud
 *    Evidence: Timing/GPIO all correct, but UART fails
 *
 * ❌ PROBLEM: Analog power still OFF after everything passed
 *    Cause: Different BQ79616 revision, different control bit
 *    Fix: Check datasheet for actual SEND_WAKE bit definition
 *    Evidence: All timing correct, communication works, power OFF
 *
 * ===========================================================================
 */
